## 🚀 Usage Example

```r
library(superassp)

# Check if DisVoice is available
if (!has_disvoice_support()) {
  install_disvoice_python()
}

# Extract F0 with voicing
f0 <- trk_dv_f0(
  "audio.wav",
  frame_shift = 10,  # 10ms = 100 Hz
  min_f0 = 75,
  max_f0 = 600,
  include_voicing = TRUE
)

# Extract formants
formants <- trk_dv_formants(
  "audio.wav",
  frame_shift = 5,   # 5ms = 200 Hz
  window_size = 25,  # 25ms window
  max_formant_freq = 5500
)

# Access results
plot(f0$tracks[, "f0"], type = "l")
plot(formants$tracks[, "F1"], type = "l", col = "blue")
lines(formants$tracks[, "F2"], col = "red")
```

## 🎓 Key Innovations

1. **In-Memory Processing**
   - Zero temporary files throughout pipeline
   - av → numpy → Parselmouth Sound → DisVoice → R
   - Significant performance improvement

2. **Lazy Loading**
   - Python modules loaded only when first used
   - No startup overhead if DisVoice not needed
   - Cached for subsequent calls (70-140x speedup)

3. **Flexible Output**
   - AsspDataObj (default, for superassp pipelines)
   - data.frame (for dplyr/tidyverse workflows)
   - list (for custom processing)

4. **Development/Production Compatibility**
   - Works in development (inst/python)
   - Works after installation (package/python)
   - Automatic path detection

5. **Proper Data Type Handling**
   - Python tuple 0-indexing
   - NumPy → R conversion
   - Parselmouth object conversion
   - TextGrid → voicing track conversion
